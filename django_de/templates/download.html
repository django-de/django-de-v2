Comment obtenir Django

Django est disponible en open-source sous la licence BSD. Il n'a aucune dépendance avec d'autres bibliothèques Python. Il y a plusieurs manières de l'obtenir :

Option 1. Prendre la dernière version de développement

La dernière version de Django est celle qui est sur le dépôt subversion (le système de contrôle des révisions). Obtenez la en utilisant cette commande shell, qui nécessite l'installation préalable de Subversion :

svn co http://code.djangoproject.com/svn/django/trunk/

Option 2. Prendre la dernière version officielle

La dernière version officielle est 0.96. Soyez conscient qu'il est possible que des parties de l'API de Django soient modifiées tant que la version 1.0 n'est pas sortie. Consultez stabilité de l'API pour obtenir des informations plus complètes. Soyez assurés que nous documentons chaque changement affectant la rétro-compatibilité de façon très détaillée pour faciliter les mises à jour.

Voici comment obtenir la version 0.96 :

Premièrement, téléchargez Django-0.96.tar.gz. Puis : 


tar xzvf Django-0.96.tar.gz
cd Django-0.96
sudo python setup.py install

Sachez que la dernière commande va automatiquement télécharger et installer setuptools si vous ne l'avez pas déjà installé. Cela requiert une connexion internet fonctionnelle.

Une fois que vous l'avez

Consulter le guide d'installation pour de plus amples explications.

Et abonnez vous à la liste de diffusion des utilisateurs de Django (anglais), où d'autres utilisateurs et les développeurs eux-mêmes font tout pour aider leurs semblables.

Quelle version est la meilleure ?

Nous améliorons Django chaque jour, et réussissons plutôt bien à conserver une certaine stabilité dans le code.

Par conséquent, utiliser la dernière version de Django est sûr et vous donne facilement accès aux dernière fonctionnalités qui ont été ajoutées. Si vous choisissez de suivre la version en développement, gardez à l'esprit qu'il peut y avoir quelques changements introduisant des incompatibilités avec les versions précédentes. Il vous faudra surveiller de près les modifications.

Si vous cherchez une version stable à déployer et si vous ne vous préoccupez pas de la prochaine version, vous préférerez utiliser la dernière version officielle (qui inclura toujours des notes détaillées sur toutes les opérations que vous aurez à faire durant la mise à jour).

Précédentes versions :

  • Django-0.95.tar.gz
  • Django-0.91.tar.gz
  • Django-0.90.tar.gz
